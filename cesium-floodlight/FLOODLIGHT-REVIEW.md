# 模型泛光实现 Review 与优化记录

## 结论

当前代码并不是“模型自发光着色器 + 模型专属泛光”，而是：

1. 使用 `S3MLayer.style3D.fillForeColor` 给模型图层整体着色；
2. 使用 `layer.brightness` 和场景级 `scene.lightSource.ambientLightColor` 提亮；
3. 对最终整张场景颜色纹理做亮度阈值提取和模糊。

因此，地形、影像、道路或其他场景内容只要亮度超过阈值，就会和模型一起发光。`isModelLayer` 只限制了 JavaScript 侧“哪些图层被改色”，并没有给后处理阶段提供模型像素掩膜。

SuperMap S3M 图层在当前 11.2.x 运行环境中没有直接暴露可供 `PostProcessStage` 使用的图层 ID 纹理；Cesium 后处理的 `selected` 机制也不能直接把整个 S3M 图层作为稳定的选区。因此，在不改动引擎或增加模型专用离屏渲染通道的前提下，本次采用“模型着色特征 + 亮度 + 深度”的软掩膜方案隔离模型。

## 发现的问题

### P0：全屏亮度提取无法保证仅模型发光

原实现的提取条件只有：

```glsl
smoothstep(threshold, threshold + 0.05, luminance) * notSky
```

`depthTexture` 只能排除天空，不能区分模型、地形和影像。阈值降低后背景必然被捕获；阈值升到 1.0 又会导致 LDR 场景中几乎没有像素可被提取。

优化：将模型的 `fillForeColor` 同时作为“着色标记”，后处理对采样颜色与该标记的归一化色相距离计算软掩膜，再与亮度和深度掩膜相乘。浏览器采样表明模型像素的色相距离主要落在 `0.04–0.08`，纯白背景约为 `0.13`，因此软掩膜使用 `0.06–0.13` 的过渡带。进一步采样显示模型像素的 `R-B` 约为 `0.11–0.17`、白色道路约为 `0.07`，所以又叠加了 `0.05–0.12` 的暖色标记软阈值，显著抑制白色道路和屋顶。

限制：这是当前引擎能力下的近似隔离。如果场景中存在与模型完全相同的高亮颜色，仍可能产生少量误捕获。严格的 100% 隔离需要引擎支持模型 ID/分类纹理，或增加一个只渲染目标 S3M 图层的离屏 mask pass。

### P0：场景级环境光会主动提亮非模型内容

`applyBrightness()` 同时写入：

```js
layer.brightness = b;
scene.lightSource.ambientLightColor = new Cesium.Color(b, b, b, 1);
```

第二项是场景级状态，与“仅模型自发光”的目标冲突，也会增加背景越过泛光阈值的概率。

优化：泛光开启时只设置目标模型图层的 `brightness`，不再修改场景环境光。

### P1：高斯归一化方式导致光晕边缘恒亮

原实现只将命中亮度掩膜的采样权重计入 `wSum`：

```glsl
bloom += source * mask * weight;
wSum += mask * weight;
```

这会把稀疏的边缘样本重新除回接近完整亮度，使光晕缺少随覆盖率递减的自然衰减，容易形成厚重、发白的块状边缘。

优化：`wSum` 累加完整高斯核权重，未命中的样本以 0 参与卷积。模糊结果会随离开模型的距离自然衰减。

### P1：光晕颜色被背景采样颜色污染

原实现累加 `sample.rgb`。误捕获的道路、屋顶等会把自己的颜色带入泛光。

优化：使用配置的模型发光色作为统一 glow color，只让掩膜控制能量。

### P1：关闭效果没有恢复图层原状态

原实现把颜色、亮度和透明排序硬编码恢复为白色、1 和 false。若图层加载时并非这些值，关闭泛光会永久改变原始显示。

优化：场景加载完成时保存每个模型图层的原始 `fillForeColor`、`brightness` 和 `orderIndependentTranslucency`，关闭时逐项恢复。

### P1：`style3D` 缺少防御性检查

图层状态尚未就绪或属性发生变化时，直接写 `layer.style3D.fillForeColor` 可能抛出异常。

优化：应用和恢复时都检查 `style3D`，单个图层失败不影响其他图层和后处理阶段。

### P2：当前 7×7 单 pass 成本较高

主泛光每像素读取 49 次颜色和深度，开启宽域后再增加一轮。当前场景和目标分辨率下可运行，但高分屏或低端 GPU 仍可能掉帧。

本次保留单 pass，以避免 SuperMap 11.2.x 多阶段/HDR 路径曾出现的 framebuffer feedback loop。后续若引擎版本升级，可改为低分辨率提取 + 水平/垂直分离高斯 + 合成。

## 本次实施策略

- 保持 HDR 关闭，规避已复现的 feedback loop。
- 仅模型图层应用填充色和亮度，不修改场景环境光。
- 使用“深度 × 亮度 × 模型色相相似度”的软掩膜。
- 修正高斯核归一化和统一光晕颜色。
- 保存并精确恢复模型图层原状态。
- 保留提取调试按钮，使其显示新的模型软掩膜。
- 在浏览器中以 `target.png` 的暖白、柔和、不过曝效果为目标调参，并把最终配置写入 `PRESETS.target`。
