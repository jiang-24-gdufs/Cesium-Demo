# 360全景与3D视角联动 — 核心逻辑技术文档

## 1. 场景定义

在 BIM 数字孪生项目中，常见需求是将**360全景照片**（室内实拍）与**3D BIM模型**并排展示，并实现**视角联动**：用户在任一视图中旋转视角，另一视图实时跟随。

```
┌─────────────────────────────────────────────────┐
│                  Browser Window                  │
├──────────────────────┬──────────────────────────┤
│   360° 全景视图       │    Cesium 3D BIM 场景    │
│                      │                          │
│   Three.js Sphere    │    3D Tiles / glTF       │
│   Camera @ 球心      │    Camera @ 拍摄点        │
│                      │                          │
│   仅 heading/pitch   │    position + heading +  │
│   + fov              │    pitch + roll           │
├──────────────────────┴──────────────────────────┤
│              SyncController                      │
│  heading/pitch 双向同步 · 防循环 · 室内约束       │
└─────────────────────────────────────────────────┘
```

### 1.1 为什么不能直接共享 Camera 对象

| 维度 | 360全景 | Cesium 3D |
|------|---------|-----------|
| 坐标系 | 球面坐标 (heading, pitch) | ECEF / WGS84 |
| 位置 | 固定在球心，不可移动 | 自由移动 |
| 投影 | 等距圆柱投影 → 球体内壁 | 透视投影 |
| 引擎 | Three.js | CesiumJS |

两者的 Camera 对象结构、坐标体系完全不同，**不能互相赋值**。正确做法是提取**公共状态量**（heading / pitch）作为同步协议。

## 3. 同步协议

### 3.1 同步状态定义

```typescript
interface SyncState {
  heading: number;  // 水平朝向角 [0°, 360°)
  pitch: number;    // 俯仰角 [-85°, 85°]（室内模式 [-60°, 60°]）
}
```

两个视图共享且**仅共享**这两个角度值。`position`、`fov`、`roll` 均由各自视图独立管理。

### 3.2 同步方向判定

```
用户操作全景视图  →  activeSource = 'panorama'  →  全景 heading/pitch → Cesium
用户操作3D视图   →  activeSource = 'cesium'    →  Cesium heading/pitch → 全景
```

通过 `pointerdown` / `wheel` 事件在各容器上注册监听，确定当前**活跃源**。只有活跃源的变化才会广播到对端。

### 3.3 防循环机制

```
全景变化 → SyncController._syncing = true
  → 设置 Cesium Camera
  → SyncController._syncing = false

Cesium preRender 触发
  → 检查 _syncing === true → 跳过（不回写全景）
```

使用 `_syncing` 标志位 + `_activeSource` 双重守卫，彻底阻断 A→B→A 死循环。

## 4. 全景视图实现（Three.js）

### 4.1 渲染原理

```
等距圆柱投影纹理 (2048×1024)
         │
         ▼
  SphereGeometry(R=500, seg=60×40)
  scale(-1, 1, 1)   ← 翻转法线，纹理贴在内壁
         │
         ▼
  MeshBasicMaterial.map = texture
         │
         ▼
  PerspectiveCamera @ (0,0,0)
  lookAt(target)     ← target 由 heading/pitch 计算
```

### 4.2 视角控制

```javascript
// heading/pitch → 球面坐标 → Camera lookAt target
const target = new THREE.Vector3(
  Math.cos(pitchRad) * Math.sin(headingRad),   // X
  Math.sin(pitchRad),                           // Y (上方向)
  Math.cos(pitchRad) * Math.cos(headingRad)     // Z
);
camera.lookAt(target);
```

鼠标拖拽增量映射：
- `dx` → `heading -= dx * 0.3`（水平灵敏度）
- `dy` → `pitch += dy * 0.3`（垂直灵敏度）
- 滚轮 → `fov += deltaY * 0.05`（缩放，范围 30°~120°）

### 4.3 关键约束

| 参数 | 普通模式 | 室内模式 |
|------|---------|---------|
| pitch 范围 | [-85°, 85°] | [-60°, 60°] |
| heading | [0°, 360°) 循环 | 同左 |
| fov | [30°, 120°] | 同左 |

室内模式收紧 pitch 的原因：在 BIM 室内全景中，极端俯仰角会看到天花板/地板的畸变区域和拼接瑕疵。

## 5. Cesium 3D 视图

### 5.1 相机状态读写

```javascript
// 读取 Cesium Camera 朝向（弧度 → 角度）
const headingDeg = Cesium.Math.toDegrees(camera.heading);
const pitchDeg   = Cesium.Math.toDegrees(camera.pitch);

// 写入 Cesium Camera 朝向（角度 → 弧度），保持 position 不变
camera.setView({
  destination: camera.positionWC.clone(),
  orientation: {
    heading: Cesium.Math.toRadians(headingDeg),
    pitch:   Cesium.Math.toRadians(pitchDeg),
    roll:    0,
  },
});
```

### 5.2 同步时机

**全景 → Cesium**：在 `PanoramaViewer.onChange` 回调中立即执行，触发频率由鼠标 `pointermove` 事件决定。

**Cesium → 全景**：在 `scene.preRender` 事件中执行，与 Cesium 渲染帧率同步（通常 60Hz）。

## 6. BIM 室内模式（CameraBounds）

### 6.1 设计目标

在 BIM 室内场景中，相机必须被限制在建筑几何体内部，防止：
- 水平方向飞出墙壁
- 垂直方向穿透天花板或地板
- 俯仰角过大导致视角异常

### 6.2 约束模型

使用一个圆柱形 AABB 约束区域：

```
        ┌─────────────────┐  ← maxHeight (天花板)
        │                 │
        │    ○ Camera     │  ← 约束在圆柱内
        │                 │
        │    ┌───┐        │
        │    │   │ radius  │  ← 水平活动半径
        │    └───┘        │
        └─────────────────┘  ← minHeight (地面)
```

### 6.3 每帧钳位逻辑

```
preRender →
  1. 读取 Camera WC position → Cartographic
  2. 计算与 center 的水平距离 (Haversine)
  3. 若 distance > radius → 按比例缩回
  4. 若 height < minHeight 或 height > maxHeight → 钳位
  5. 若 pitch 超出 [pitchMin, pitchMax] → 钳位
  6. 若任一条件触发 → camera.setView(clampedValues)
```

### 6.4 配置参数

```javascript
{
  center: {
    longitude: 116.3912,   // 全景拍摄点经度
    latitude: 39.9060,     // 全景拍摄点纬度
    height: 5              // 拍摄高度（楼层标高）
  },
  radius: 30,              // 水平活动半径(米) — 对应房间大小
  minHeight: 1,            // 最低高度(米) — 地面以上
  maxHeight: 15,           // 最高高度(米) — 天花板以下
  pitchMin: -60,           // 最小俯仰角(度)
  pitchMax: 60             // 最大俯仰角(度)
}
```

### 6.5 边界可视化

通过 Cesium Entity 渲染约束边界，辅助调试：
- **圆柱体**：半透明青色，显示水平活动范围
- **底面椭圆**：地面边界
- **顶面椭圆**：天花板边界

## 7. 同步流程时序图

### 7.1 全景驱动（用户操作全景）

```
用户 pointerdown → activeSource = 'panorama'
用户 pointermove →
  PanoramaViewer:
    heading -= dx * 0.3
    pitch += dy * 0.3
    pitch = clamp(pitch, pitchMin, pitchMax)
    updateCamera(lookAt)
    fireChange('panorama')
      │
      └→ SyncController.onChange:
           检查: syncEnabled? ✓  syncing? ✗  source='panorama'? ✓
           _syncing = true
           Cesium.camera.setView(heading, pitch)  ← position 不变
           _syncing = false
      │
      └→ Cesium preRender:
           检查: source='panorama' ≠ 'cesium' → 跳过（不回写）
```

### 7.2 Cesium 驱动（用户操作 3D 场景）

```
用户 pointerdown → activeSource = 'cesium'
用户拖拽 Cesium →
  Cesium Camera 自动更新 heading/pitch/position
      │
      └→ scene.preRender:
           检查: syncEnabled? ✓  syncing? ✗  source='cesium'? ✓
           _syncing = true
           headingDeg = toDegrees(camera.heading)
           pitchDeg = toDegrees(camera.pitch)
           PanoramaViewer.setView(headingDeg, pitchDeg)
           _syncing = false
```

### 7.3 室内模式下的附加约束

```
scene.preRender (CameraBounds):
  读取 Camera position →
  Haversine(camera, center) > radius?
    → 缩回到边界
  height 超出 [min, max]?
    → 钳位
  pitch 超出 [pitchMin, pitchMax]?
    → 钳位
  → camera.setView(clamped)
```

## 10. 关键设计决策与权衡

### 10.1 为什么只同步 heading/pitch，不同步 position？

360全景的相机**没有真实空间位置**——它固定在球心。如果强行同步 position，全景视图无法响应（球心不能移动），而 Cesium 侧的 position 变化也不应反映到全景中。heading/pitch 是两者唯一的**公共语义**。

### 10.2 为什么用 preRender 而不是 postRender 或 requestAnimationFrame？

- `preRender`：在 Cesium 计算当前帧的渲染命令**之前**触发，此时修改 Camera 能在**同一帧**生效，无延迟
- `postRender`：帧已渲染完毕，修改 Camera 要到**下一帧**才能看到，产生一帧延迟
- `requestAnimationFrame`：独立于 Cesium 渲染循环，可能与 Cesium 帧不对齐

### 10.3 室内约束为什么用 Haversine 而不是 Cartesian3 距离？

Cesium Camera 的 `positionWC` 是 ECEF 坐标，直接用 Cartesian3 距离计算的是三维空间的直线距离，不是地表水平距离。在室内场景中，我们关心的是**水平面内**的活动半径，Haversine 公式给出的是地表弧面距离，在小范围内（<100m）近似等于水平距离，语义更准确。

### 10.4 防循环为什么需要 syncing + activeSource 双重守卫？

单靠 `_syncing`：
```
用户拖拽全景 → syncing=true → 设置 Cesium → syncing=false
Cesium preRender → syncing=false → 读取 Cesium → 设置全景 ← 这里会回写！
```

加上 `activeSource` 判断后，只要用户还在操作全景（`activeSource='panorama'`），Cesium→全景方向的同步就被跳过，彻底阻断循环。

## 11. 扩展方向

| 方向 | 说明 |
|------|------|
| 真实全景纹理 | 替换程序生成的 Canvas 纹理为实拍 equirectangular 图片 |
| 多拍摄点切换 | 配置多个全景点位，点击切换时 Cesium 同步 flyTo 对应位置 |
| BIM 构件拾取联动 | 点击全景中的热点 → 高亮 Cesium 中对应的 BIM 构件 |
| pitch 插值平滑 | 使用 lerp 做状态插值，消除同步时的跳变感 |
| 多楼层支持 | CameraBounds 按楼层切换 minHeight/maxHeight |
| Web Worker 解码 | 大尺寸全景图（8K+）的纹理解码移入 Worker |
| FOV 同步 | 将全景 FOV 映射到 Cesium Camera frustum.fov |
