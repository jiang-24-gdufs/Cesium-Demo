# Cesium + OpenLayers 二三维联动 Demo 实现文档

## 一、整体架构概述

本 Demo 实现了 **Cesium 三维场景** 与 **OpenLayers 二维 MVT 矢量瓦片地图** 的双屏联动，核心功能包括：

1. **拾取联动**：三维场景拾取对象后联动高亮二维 MVT 要素；反之亦然
2. **活跃源追踪**：检测用户当前操作的是三维还是二维面板
3. **S3M 模型高亮管理**：统一管理三维 S3MTilesLayer 的选中/取消选中
4. **图层管理**：三维 S3M 图层和二维 MVT 图层的可见性控制与定位

> **重要限制**：二维拾取联动三维时，**仅做高亮不做相机飞行定位**。原因是 BIM 3D 数据集不返回可用的空间坐标字段（SMSDRI*、geometry、SMX/SMY 均为空），且 S3MTilesLayer 没有 `getFeatureBounds(smId)` 公开 API。详细的方案探索和失败分析参见 [二维联动三维定位方案总结](./二维联动三维定位方案总结.md)。

### 2.3 前置数据要求

1. **三维场景**：iServer 发布的 realspace 场景服务，包含多个 S3MTilesLayer（BIM 模型）
2. **MVT 服务**：iServer 发布的矢量瓦片地图服务，需提供：
   - `{baseUrl}.json` — 地图元数据（坐标系、范围、比例尺等）
   - `{baseUrl}/style.json` — Mapbox Style 规范的渲染样式
   - `{baseUrl}/tiles/{z}/{x}/{y}.mvt` — MVT 瓦片数据
3. **数据服务**：iServer 数据服务（REST Data），用于属性查询，数据集需包含：
   - `UNIQUEID` 字段 — 跨二三维的联动 Key（UUID 格式，如 `625e94e5-bd28-46d9-aa3b-687482920dfa-00068523`）
   - `SMID` 字段 — S3M 图层内对象标识
   - `SMMAXZ/SMMINZ` — 构件高度范围（BIM 数据集**仅返回**这两个空间字段）
   - 完整的字段样例参见 [feature.fields.md](./feature.fields.md)
4. **坐标系**：二维使用 **EPSG:3857**（Web 墨卡托），三维使用 **WGS84**

## 四、联动调用逻辑详解

### 4.1 三维拾取 → 联动二维（3D → 2D）

```
用户点击三维场景
    ↓
ScreenSpaceEventHandler LEFT_CLICK
    ↓
viewer3D.scene.pick(position) 获取 picked 对象
    ↓
提取 layer（S3MTilesLayer）和 smId
    ↓
syncController.highlightS3MObject(layer, [smId])  ← 三维高亮
    ↓
matchDataset(layer) → 匹配数据集名
    ↓
doSqlQuery(DATA_URL, "datasource:dataset", "SmID = " + smId)
    ↓
获取属性信息 → showInfoFromObject(info, '三维拾取')
    ↓
linkPick3Dto2D(pickedCartesian, info, smId)  ← 联动二维
    ├── 提取 UNIQUEID
    ├── 遍历所有 DATASETS，用 "UNIQUEID = 'xxx'" 过滤查询
    │       ↓ 命中 → highlightedAttr = { name: 'UniqueID', value }
    │       ↓ highlightLayer.changed() → 触发 MVT 高亮重绘
    │       ↓ findAndOutlineMVTFeature() → 在已加载瓦片中查找并绘制精确轮廓线
    └── 未命中 → toast 提示 "该BIM构件没有对应的图元"
```

### 4.2 二维拾取 → 联动三维（2D → 3D）

```
用户点击二维地图
    ↓
map2D 'click' 事件 → onMap2DClick(evt)
    ↓
map2D.forEachFeatureAtPixel(pixel, callback, { hitTolerance: 5 })
    ↓
获取 MVT Feature → feat.getProperties()
    ↓
highlightedFeatureId = feat.getId()
highlightLayer.changed()  ← 二维 MVT 高亮
showOutlineForFeature(feat)  ← 绘制轮廓线
    ↓
showInfoFromObject(props, '二维拾取 (MVT)')
    ↓
extractLinkKey(props) → 提取 UniqueID
    ↓
highlightInModelByKey(uniqueId)  ← 联动三维
    ├── 遍历所有 DATASETS（15个）
    ├── doSqlQuery 用 "UNIQUEID = 'xxx'" 查询
    ├── 找到后调用 doHighlight3D(feat, ds, linkKey)
    │       ↓ extractSmId(feat)
    │       ↓ matchS3MLayerByDataset(ds) → 匹配目标 S3M 图层
    │       ↓ syncController.highlightS3MObject(layer, [smId])  ← 三维高亮（仅高亮，不飞行）
    └── 全部未命中 → toast "该图元没有对应的BIM构件"
```

> **注意**：二维→三维的联动**仅高亮**目标构件，不执行 `camera.flyTo` 定位。这是经过多轮验证后的设计决策，详见 [二维联动三维定位方案总结.md](./二维联动三维定位方案总结.md) 第 9 节。

### 4.3 联动 Key —— UniqueID

跨二三维联动的唯一标识字段为 `UNIQUEID`：

- **类型**：字符串（UUID 格式），如 `625e94e5-bd28-46d9-aa3b-687482920dfa-00068523`
- **来源**：BIM 模型中每个构件的全局唯一标识，在三维数据集和二维 MVT 要素中均存在
- **提取**：`extractLinkKey(info)` 函数从属性中按大小写变体查找 `UNIQUEID`/`UniqueId`/`uniqueid`/`UniqueID`
- **SQL 查询**：`"UNIQUEID = 'xxx'"` — 注意值需要用单引号包裹（字符串类型）

### 4.4 二维 MVT 高亮机制

不使用 OpenLayers 内置的 selection/interaction，而是采用**双图层叠加**方案：

1. **高亮图层**（`highlightLayer`）：独立 `VectorTile` 图层，共享 MVT source，自定义 style function：
   - 通过 `highlightedFeatureId`（feature ID 匹配）或 `highlightedAttr`（属性名/值匹配）判断
   - 匹配 → 返回黄色高亮样式（`#FFE600` 描边 + `rgba(255, 230, 0, 0.3)` 填充）
   - 不匹配 → 返回 `null`（不渲染）
   - 触发刷新：`highlightLayer.changed()`

2. **轮廓图层**（`outlineLayer`）：独立 `Vector` 图层，通过 `findAndOutlineMVTFeature()` 从已加载瓦片中查找匹配要素的几何，克隆后添加到轮廓 source：
   - 黄色虚线描边（`#FFE600`，`lineDash: [6, 4]`）
   - `zIndex: 999` 确保在最上层

这种方式的优势是无需重新请求瓦片数据，只需重新执行 style function。

## 六、二维 MVT 地图初始化

### 6.1 初始化流程

```
fetch(MVT_BASE_URL + '.json')          → 获取地图元数据 (originResult)
    ↓
fetch(MVT_STYLE_URL)                    → 获取 style.json
    ↓
buildMVTMap()                           → 创建地图和 MVT 图层
```

失败时调用 `createFallbackMap()` 创建空白地图，确保联动控制器仍可初始化。

### 6.2 关键参数计算

#### 分辨率数组 (styleResolutions)

MVT 渲染需要完整的分辨率数组（22 级），从地图范围计算：

```javascript
function getStyleResolutions(bounds) {
  var temp = Math.abs(bounds.left - bounds.right) / 512;
  for (var i = 0; i < 22; i++) {
    styleResolutions[i] = temp;
    temp = temp / 2;
  }
}
```

#### 视图分辨率 (visableResolution)

从服务配置的可见比例尺转换，或使用硬编码的 6 级：

```javascript
[2.389, 1.194, 0.597, 0.299, 0.149, 0.075]
```

#### minZoom 计算

将视图的第一级分辨率与 styleResolutions 对比，找到最接近的索引作为 `minZoom`。

### 6.3 MVT 图层创建

```javascript
var format = new ol.format.MVT({ featureClass: ol.Feature });
var mbStyle = new ol.supermap.MapboxStyles({
  style: styleJson,
  source: styleJson.name,
  resolutions: styleResolutions,
  map: map2D,
});

mbStyle.on('styleloaded', function () {
  mvtLayer = new ol.layer.VectorTile({
    declutter: true,
    source: new ol.source.VectorTile({
      projection: 'EPSG:3857',
      url: MVT_TILE_URL,
      tileGrid: new ol.tilegrid.TileGrid({
        resolutions: styleResolutions,
        origin: [envelope.left, envelope.top],
        minZoom: minZoom,
        tileSize: 512,
      }),
      format: format,
    }),
    style: mbStyle.getStyleFunction(),
  });
  map2D.addLayer(mvtLayer);
});
```

## 七、属性查询（iServer REST Data）

### 7.1 查询方法封装

```javascript
function doSqlQuery(serviceUrl, datasetName, filter, onSuccess, options)
```

- 使用 `SuperMap.REST.GetFeaturesBySQLService`（[API 清单](./SuperMap.REST.keys.md)）
- `options.hasGeometry`：控制是否返回几何信息（BIM 数据集即使设为 `true` 也返回 `null`）
- `options.toIndex`：控制最大返回数量（默认 49，即最多 50 条）
- 失败时静默调用 `onSuccess([])`，不抛出异常

### 7.2 查询场景

| 场景 | filter 示例 | 目的 |
|------|-------------|------|
| 三维拾取后查属性 | `"SmID = 42"` | 获取对象的完整属性字段 |
| 二维联动三维高亮 | `"UNIQUEID = 'xxx'"` | 找到三维对应对象的 SmID |
| 三维联动二维高亮 | `"UNIQUEID = 'xxx'"` | 确认二维数据集中存在对应要素 |

### 7.3 跨数据集遍历查询

联动查询时需要**遍历所有 15 个 DATASETS**，对每个执行 SQL 查询，找到第一个命中的即停止（通过 `found` 标记位）。已发出的其余请求仍会完成但回调被跳过。

## 八、图层管理逻辑

### 8.1 三维图层

- 来源：`viewer3D.scene.open()` 返回的 `layers[]`
- 可见性：`layer.visible = true/false`
- 定位策略（优先级）：
  1. `layer._boundingSphere`（center 非 ZERO）→ `camera.flyToBoundingSphere`
  2. `layer._layerBounds`（有 west 属性）→ `camera.flyTo(Rectangle)`
  3. `layer.lon/lat/height`（positionUnits 为 Degree 且值有效）→ `camera.flyTo(Cartesian3)`
- 显示名称：`getS3MDisplayName(layer, index)` — 去掉项目后缀和路径信息

### 8.2 二维图层

- 来源：MVT 图层实例 `mvtLayer`
- 可见性：`layer.setVisible(bool)`
- 定位：`view.fit(extent, { padding })` — 从 `mvtOriginResult.bounds` 提取范围

### 8.3 图层面板

- 三维图层支持全选/全不选（master checkbox）
- 每个图层项包含：可见性复选框 + 名称 + 定位按钮
- 二维图层额外显示 `MVT` 类型徽章

## 十一、关键辅助函数

### 11.1 图层名与数据集名匹配

```javascript
function matchDataset(layer) {
  var layerName = (layer._name || layer.name || '').toLowerCase();
  for (var i = 0; i < DATASETS.length; i++) {
    var dsClean = DATASETS[i].replace('_dongchesuo', '').toLowerCase();
    if (layerName.indexOf(dsClean) !== -1 || dsClean.indexOf(layerName) !== -1) return ds;
  }
}
```

采用双向包含匹配，去掉项目后缀后比对。匹配同一逻辑也用于 `matchS3MLayerByDataset(datasetName)`（反向：从数据集名找图层）。

### 11.3 墨卡托坐标校验

```javascript
function isValidMercatorCoord(x, y) {
  return Math.abs(x) > 100000 && Math.abs(y) > 100000;
}
```
