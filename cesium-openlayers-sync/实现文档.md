# Cesium + OpenLayers 二三维联动 Demo 实现文档

## 一、整体架构概述

本 Demo 实现了 **Cesium 三维场景** 与 **OpenLayers 二维 MVT 矢量瓦片地图** 的双屏联动，核心功能包括：

1. **拾取联动**：三维场景拾取对象后联动定位并高亮二维 MVT 要素；反之亦然
2. **活跃源追踪**：检测用户当前操作的是三维还是二维面板
3. **S3M 模型高亮管理**：统一管理三维 S3MTilesLayer 的选中/取消选中
4. **图层管理**：三维 S3M 图层和二维 MVT 图层的可见性控制与定位

### 文件结构

| 文件 | 职责 |
|------|------|
| `index.html` | 页面结构，双屏布局，引入依赖库 |
| `sync-controller.js` | 联动控制器核心类 `CesiumOpenLayersSyncController` |
| `app.js` | 业务逻辑主文件（初始化、拾取、查询、联动调用） |
| `style.css` | 全部 UI 样式 |
| `iServer.html` | iServer 原始 MVT 预览页面（作为参考实现） |

### 2.2 后端服务

| 服务 | URL | 用途 |
|------|-----|------|
| 三维模型场景 | `https://ct.sunrtcloud.com/iserver/services/3D-dongchesuotest/rest/realspace` | S3M 三维模型瓦片 |
| MVT 矢量瓦片 | `https://ct.sunrtcloud.com/iserver/services/map-mvt-dongchesuo/restjsr/v1/vectortile/maps/dongchesuo` | 二维矢量瓦片数据 |
| 数据服务 | `https://ct.sunrtcloud.com/iserver/services/data-dongchesuotest/rest/data` | 属性查询（GetFeaturesBySQL） |

### 2.3 前置数据要求

1. **三维场景**：iServer 发布的 realspace 场景服务，包含多个 S3MTilesLayer（BIM 模型）
2. **MVT 服务**：iServer 发布的矢量瓦片地图服务，需提供：
   - `{baseUrl}.json` — 地图元数据（坐标系、范围、比例尺等）
   - `{baseUrl}/style.json` — Mapbox Style 规范的渲染样式
   - `{baseUrl}/tiles/{z}/{x}/{y}.mvt` — MVT 瓦片数据
3. **数据服务**：iServer 数据服务（REST Data），用于属性查询，数据集需包含：
   - `ELEMENTID` 字段 — 跨二三维的联动 Key
   - `SMID` 字段 — S3M 图层内对象标识
   - `SMX/SMY/SMZ` 或 `SMSDRIW/SMSDRIE/SMSDRIN/SMSDRIS` — 墨卡托坐标范围字段
4. **坐标系**：二维使用 **EPSG:3857**（Web 墨卡托），三维使用 **WGS84**，联动时需要互转

### 3.2 职责与内部状态

| 属性 | 类型 | 说明 |

### 3.3 核心方法

#### 活跃源追踪

```javascript
_setupSourceTracking()
```

- 在 Cesium 容器和 OL 容器上分别监听 `pointerdown` 和 `wheel` 事件
- 触发时更新 `_activeSource` 并回调所有注册的监听器
- 通过 `onActiveSourceChange(callback)` 注册回调

#### 二维拾取标记

```javascript
_setupPickOverlay()          // 创建 ol.Overlay + DOM 元素
showPickMarker2D(coordinate) // 显示标记在指定墨卡托坐标
clearPickMarker2D()          // 隐藏标记
```

- 使用 `ol.Overlay` 实现，DOM 元素类名 `.ol-pick-overlay`
- CSS 动画实现脉冲效果

#### 三维拾取标记

```javascript
showPickEntity3D(lng, lat, height, name) // 在经纬度位置添加 Point + Label
clearPickEntity3D()                       // 移除 Entity
```

- 使用 `Cesium.Entity` 的 `point` + `label` 组合
- 设置 `disableDepthTestDistance: Number.POSITIVE_INFINITY` 确保始终可见

#### 三维 S3M 高亮管理

```javascript
highlightS3MObject(layer, smIds) // 高亮指定图层的指定 SmID
clearS3MHighlight(allLayers?)    // 清除高亮
```

- 底层调用 `layer.selectedColor = color` + `layer.setSelection(smIds)`
- 清除调用 `layer.releaseSelection()`
- 自动管理"先清除旧高亮再设置新高亮"的逻辑

#### 精确要素定位

```javascript
flyToFeatureBounds(fields, mercatorToLonLat, linkKey) → boolean
```

- 从要素的 `SMSDRIW/SMSDRIE/SMSDRIN/SMSDRIS` + `SMMINZ/SMMAXZ` 计算精确包围盒
- 墨卡托坐标通过 `mercatorToLonLat` 转为经纬度
- 构造 `Cesium.BoundingSphere` 并调用 `camera.flyToBoundingSphere`
- 相机距离 = `radius * 3.0`，俯仰角 `-40°`

## 四、联动调用逻辑详解

### 4.1 三维拾取 → 联动二维（3D → 2D）

```
用户点击三维场景
    ↓
ScreenSpaceEventHandler LEFT_CLICK
    ↓
viewer3D.scene.pick(position) 获取 picked 对象
    ↓
提取 layer（S3MTilesLayer）和 smId
    ↓
syncController.highlightS3MObject(layer, [smId])  ← 三维高亮
    ↓
matchDataset(layer) → 匹配数据集名
    ↓
doSqlQuery(DATA_URL, "datasource:dataset", "SmID = " + smId)
    ↓ 
获取属性信息 → showInfoFromObject(info, '三维拾取')
    ↓
linkPick3Dto2D(screenPos, info, smId)  ← 联动二维
    ├── 提取 ELEMENTID
    ├── queryFeatureCoordFor2D(elementId, callback)
    │       ↓ 遍历所有数据集，用 "ELEMENTID = xxx" 过滤查询
    │       ↓ 获取要素的墨卡托坐标 → callback([x, y])
    │       ↓
    ├── flyOLToCoord(olCoord)  ← OL 视角动画飞行
    │       ↓ view.animate({ center, zoom, duration: 800 })
    │       ↓ syncController.showPickMarker2D(olCoord)
    │       ↓
    └── highlightMVTFeatureByAttr('ElementID', elementId)  ← MVT 高亮
            ↓ 设置 highlightedAttr = { name, value }
            ↓ highlightLayer.changed() → 触发重绘
            ↓ style 函数中比对属性值决定是否返回高亮样式
```

**Fallback 路径**：如果数据服务查询不到坐标，则使用 `viewer3D.scene.pickPosition(screenPos)` 获取三维屏幕坐标对应的世界坐标，再通过 `ol.proj.fromLonLat` 转为墨卡托坐标定位 OL。

### 4.2 二维拾取 → 联动三维（2D → 3D）

```
用户点击二维地图
    ↓
map2D 'click' 事件 → onMap2DClick(evt)
    ↓
syncController.showPickMarker2D(evt.coordinate)  ← 二维标记
    ↓
map2D.forEachFeatureAtPixel(pixel, callback, { hitTolerance: 5 })
    ↓
获取 MVT Feature → feat.getProperties()
    ↓
highlightedFeatureId = feat.getId()
highlightLayer.changed()  ← 二维 MVT 高亮
    ↓
showInfoFromObject(props, '二维拾取 (MVT)')
    ↓
extractLinkKey(props) → 提取联动 Key
    ├── 优先 ElementID
    ├── 其次 UNIQUEID
    └── 从 BlockName 解析 (_dwg-{数字}-)
    ↓
highlightInModelByKey(linkKey)  ← 联动三维
    ├── 遍历所有 DATASETS
    ├── doSqlQuery 用 "ELEMENTID = xxx" 查询
    ├── 找到后调用 doHighlight3D(feat, ds, linkKey)
    │       ↓ extractSmId(feat)
    │       ↓ 匹配目标 S3M 图层
    │       ↓ syncController.highlightS3MObject(layer, [smId])  ← 三维高亮
    │       ↓ syncController.flyToFeatureBounds(fields, mercatorToLonLat, linkKey)
    │       ↓  或 fallback: camera.flyTo(经纬度坐标)
    │       ↓ syncController.showPickEntity3D(lng, lat, h, label)
    └── 更新状态栏
```

### 4.3 联动 Key 匹配策略

联动的核心在于 **跨二三维的统一标识字段**：

| 优先级 | 字段 | 说明 |
|--------|------|------|
| 1 | `ELEMENTID` | 数值型，BIM 模型中的元素唯一 ID，跨二三维一致 |
| 2 | `UNIQUEID` | 字符串型唯一标识 |
| 3 | `BlockName` 解析 | 从 `_dwg-{number}-` 模式中提取 ElementID |
| 4 | `SmID` | SuperMap 内部对象 ID（仅在同一数据源内有意义） |

### 4.4 坐标转换

| 转换方向 | 方法 | 使用场景 |
|----------|------|----------|
| 墨卡托 → 经纬度 | `mercatorToLonLat(mx, my)` | 数据服务返回的坐标转为 Cesium 所需的 WGS84 |
| 经纬度 → 墨卡托 | `ol.proj.fromLonLat([lng, lat])` | Cesium 坐标转为 OL 所需的 EPSG:3857 |
| 屏幕坐标 → 世界坐标 | `viewer3D.scene.pickPosition(screenPos)` | 三维 pick 时获取世界坐标 |
| Cartesian3 → 经纬度 | `Cesium.Cartographic.fromCartesian(cartesian)` | 三维世界坐标转 WGS84 |

`mercatorToLonLat` 实现：

```javascript
function mercatorToLonLat(mx, my) {
  var lon = (mx / 20037508.342789244) * 180;
  var lat = (my / 20037508.342789244) * 180;
  lat = (180 / Math.PI) * (2 * Math.atan(Math.exp(lat * Math.PI / 180)) - Math.PI / 2);
  return [lon, lat];
}
```

### 6.2 关键参数计算

#### 分辨率数组 (styleResolutions)

MVT 渲染需要完整的分辨率数组（22 级），从地图范围计算：

```javascript
function getStyleResolutions(bounds) {
  var temp = Math.abs(bounds.left - bounds.right) / 512;
  for (var i = 0; i < 22; i++) {
    styleResolutions[i] = temp;
    temp = temp / 2;
  }
}
```

#### 视图分辨率 (visableResolution)

服务配置的可见比例尺转为分辨率：

```javascript
resolution = 1 / (scale * 96 * inchPerMeter * meterPerMapUnit)
```

服务直接提供了 6 级：`[2.389, 1.194, 0.597, 0.299, 0.149, 0.075]`

将视图的第一级分辨率与 styleResolutions 对比，找到最接近的索引：

```javascript
function getCorrectZoom(visableResolution, styleResolutions) {
  // 找到 styleResolutions 中与 visableResolution 差值最小的索引
  // 如果该分辨率大于目标，取下一级
}
```

### 6.3 MVT 图层创建

```javascript
var format = new ol.format.MVT({ featureClass: ol.Feature });
format.defaultDataProjection = new ol.proj.Projection({
  code: 'EPSG:3857',
  units: ol.proj.Units.TILE_PIXELS,
});

var mbStyle = new ol.supermap.MapboxStyles({
  style: styleJson,
  source: styleJson.name,
  resolutions: styleResolutions,
  map: map2D,
});

mbStyle.on('styleloaded', function () {
  mvtLayer = new ol.layer.VectorTile({
    declutter: true,
    source: new ol.source.VectorTile({
      projection: 'EPSG:3857',
      url: MVT_TILE_URL,
      wrapX: false,
      tileGrid: new ol.tilegrid.TileGrid({
        resolutions: styleResolutions,
        origin: [envelope.left, envelope.top],
        minZoom: minZoom,
        tileSize: 512,
      }),
      format: format,
    }),
    style: mbStyle.getStyleFunction(),
  });
  map2D.addLayer(mvtLayer);
});
```

### 6.4 高亮图层

独立的 `VectorTile` 图层，共享 MVT source，自定义 style function：

```javascript
highlightLayer = new ol.layer.VectorTile({
  source: mvtLayer.getSource(),
  declutter: true,
  style: function (feature) {
    // 通过 highlightedFeatureId 或 highlightedAttr 匹配

      return new ol.style.Style({
```

## 七、属性查询（iServer REST Data）

### 7.1 查询方法封装

```javascript
function doSqlQuery(serviceUrl, datasetName, filter, onSuccess) {
  var getFeatureParam = new SuperMap.REST.FilterParameter({
    attributeFilter: filter,      // SQL WHERE 子句，如 "ELEMENTID = 123"
  });
  var params = new SuperMap.REST.GetFeaturesBySQLParameters({
    queryParameter: getFeatureParam,
    toIndex: 49,                   // 最多返回 50 条
    datasetNames: [datasetName],   // 格式: "datasource:dataset"
    returnContent: true,
  });
  var service = new SuperMap.REST.GetFeaturesBySQLService(serviceUrl, {
    eventListeners: {
      processCompleted: function (resultSet) { /* 处理结果 */ },
      processFailed: function (err) { /* 处理失败 */ },
    },
  });
  service.processAsync(params);
}
```

### 7.2 查询场景

| 场景 | filter 示例 | 目的 |
|------|-------------|------|
| 三维拾取后查属性 | `"SmID = 42"` | 获取对象的完整属性字段 |
| 二维联动三维 | `"ELEMENTID = 12345"` | 找到三维对应对象的 SmID 和坐标 |
| 三维联动二维定位 | `"ELEMENTID = 12345"` | 获取墨卡托坐标用于 OL 定位 |

## 十一、关键技术细节

### 11.1 MVT 高亮实现方式

不使用 OpenLayers 内置的 selection/interaction，而是创建一个**叠加的 VectorTile 图层**，共享同一 source，通过自定义 style function 决定渲染：

- 匹配的 feature → 返回高亮样式
- 不匹配 → 返回 `null`（不渲染）
- 触发刷新：`highlightLayer.changed()`

这种方式的优势是无需重新请求瓦片数据，只需重新执行 style function。

### 11.2 S3M 选中机制

SuperMap S3MTilesLayer 提供：

- `layer.selectedColor = Cesium.Color` — 设置选中颜色
- `layer.setSelection([smId1, smId2, ...])` — 选中指定对象
- `layer.releaseSelection()` — 释放所有选中

### 11.3 跨数据集遍历查询

由于三维场景包含多个 S3M 图层对应不同数据集，联动查询时需要**遍历所有 DATASETS**（15个），对每个执行 SQL 查询，找到第一个命中的即停止（通过 `found` 标记位）。

### 11.4 定位精度策略

| 优先级 | 方法 | 精度 |
|--------|------|------|
| 1 | SMSDRI 范围字段构造 BoundingSphere | 要素级别精确 |
| 2 | SMX/SMY 中心点 + 固定高度偏移 | 要素中心 |
| 3 | 图层 _boundingSphere | 图层级别 |
| 4 | Cesium.pickPosition 屏幕坐标反算 | 像素级 |

### 11.5 图层名与数据集名匹配

```javascript
function matchDataset(layer) {
  var layerName = (layer._name || layer.name || '').toLowerCase();
  for (var i = 0; i < DATASETS.length; i++) {
    var dsClean = DATASETS[i].replace('_dongchesuo', '').toLowerCase();
    if (layerName.indexOf(dsClean) !== -1 || dsClean.indexOf(layerName) !== -1) {
      return DATASETS[i];
    }
  }
}
```

采用双向包含匹配，去掉项目后缀后比对。
