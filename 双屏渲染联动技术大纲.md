# CesiumJS 双屏对等联动技术大纲

## 1. 项目概述

基于 CesiumJS 实现同一页面内两个**完全对等**的 Viewer 实例的双屏渲染，并在相机视角、时钟、图层、实体等方面实现**双边联动**（Peer-to-Peer Sync）。

### 1.1 核心原则

- **对等（Peer）架构**：两个 Viewer 地位完全相同，不存在主控/从属之分
- **操作即主导**：用户正在交互的 Viewer 自动成为同步源（Source），另一端为同步目标（Target），角色随交互实时切换
- **独立可控**：每个 Viewer 拥有独立的场景模式、影像图层控制，同步开关可随时启用/禁用
- **防循环**：通过源端锁定机制（而非简单标志位）杜绝 A→B→A 的递归同步

## 3. 核心架构

### 3.1 对等双 Viewer 架构

```
┌──────────────────────────────────────────────────┐
│                  Browser Window                   │
├──────────────────────┬───────────────────────────┤
│    Viewer A (Peer)   │     Viewer B (Peer)       │
│   - 独立场景模式     │   - 独立场景模式          │
│   - 独立影像图层     │   - 独立影像图层          │
│   - 独立 Canvas      │   - 独立 Canvas           │
│   - 可成为 Source    │   - 可成为 Source         │
│     或 Target        │     或 Target             │
├──────────────────────┴───────────────────────────┤
│              PeerSyncController                   │
│  ┌─────────────────────────────────────────────┐ │
│  │  ActiveSourceTracker                        │ │
│  │  - pointer/wheel 事件追踪                   │ │
│  │  - 确定当前 Source Viewer                   │ │
│  └─────────────────────────────────────────────┘ │
│  ┌────────────┬──────────────┬─────────────────┐ │
│  │ CameraSync │  ClockSync   │  EntitySync     │ │
│  │ 双向相机   │  双向时钟    │  双向实体选中   │ │
│  │ 同步       │  同步        │  联动           │ │
│  └────────────┴──────────────┴─────────────────┘ │
│  ┌─────────────────────────────────────────────┐ │
│  │  LayerSync (可选)                           │ │
│  │  - 影像图层可独立也可联动切换               │ │
│  └─────────────────────────────────────────────┘ │
└──────────────────────────────────────────────────┘
```

### 3.2 同步方向决策流程

```
用户交互 → ActiveSourceTracker 捕获事件
          → 判定活跃 Viewer (A 或 B)
          → Source = 活跃端, Target = 对端
          → 各 Sync 模块执行 Source → Target 同步
          → 同步期间锁定 Target 的反向同步
```

### 3.3 关键限制

- CesiumJS **不支持**单 Canvas 多视口，必须创建两个独立 Viewer 实例
- Entity/Primitive 实例**不能跨 Viewer 共享**，需分别创建
- GPU 资源在两个 Canvas 之间完全独立，会产生双倍显存开销

## 4. 功能模块

### 4.1 双屏布局

- 左右分屏（默认 50/50），支持拖拽调整比例
- 响应式布局适配不同屏幕尺寸
- 可折叠/展开单侧视图
- 两个面板标签对等（Viewer A / Viewer B），无主从暗示

### 4.2 活跃源追踪 (ActiveSourceTracker)

**核心职责：** 追踪用户当前正在交互的 Viewer，决定同步方向。

**监听事件：**
- `pointerdown` — 用户开始拖拽/点击
- `wheel` — 用户滚轮缩放
- `pointermove`（可选，仅在 pointerdown 后）— 拖拽过程中确认

**状态模型：**
- `activeSource`: `'A'` | `'B'` | `null`
- 当用户在某个 Viewer 上交互时，设为该端
- 当用户无操作（idle）时，保持上次状态

### 4.3 相机联动 (CameraSync)

**双向对等同步：** 通过 `scene.preRender` 事件在每帧检测相机变化，从 activeSource 同步到对端。

```javascript
class PeerSyncController {
  _setupCameraSync() {
    // A 和 B 都注册 preRender 监听
    this._viewerA.scene.preRender.addEventListener(() => {
      if (!this._cameraSyncEnabled || this._activeSource !== 'A') return;
      this._syncCamera(this._viewerA, this._viewerB);
    });

    this._viewerB.scene.preRender.addEventListener(() => {
      if (!this._cameraSyncEnabled || this._activeSource !== 'B') return;
      this._syncCamera(this._viewerB, this._viewerA);
    });
  }

  _syncCamera(source, target) {
    if (this._syncing || target.scene.mode === Cesium.SceneMode.MORPHING) return;
    this._syncing = true;
    try {
      target.camera.setView({
        destination: source.camera.positionWC.clone(),
        orientation: {
          heading: source.camera.heading,
          pitch: source.camera.pitch,
          roll: source.camera.roll,
        },
      });
    } finally {
      this._syncing = false;
    }
  }
}
```

**防循环机制：**
- `_syncing` 标志位：在 `setView` 执行期间阻断反向监听
- `_activeSource` 门控：仅当该端为活跃源时才触发同步，天然防止循环

### 4.4 时钟联动 (ClockSync) — 双向

**与旧版主从单向的区别：** 任一端的时钟操作都应同步到对端。

```javascript
_setupClockSync() {
  const syncClock = (sourceClock, targetClock) => {
    if (!this._clockSyncEnabled || this._clockSyncing) return;
    this._clockSyncing = true;
    try {
      if (!Cesium.JulianDate.equals(targetClock.currentTime, sourceClock.currentTime)) {
        targetClock.currentTime = Cesium.JulianDate.clone(sourceClock.currentTime);
      }
      targetClock.multiplier = sourceClock.multiplier;
      targetClock.shouldAnimate = sourceClock.shouldAnimate;
    } finally {
      this._clockSyncing = false;
    }
  };

  this._viewerA.clock.onTick.addEventListener((clock) => {
    if (this._activeSource === 'A') {
      syncClock(clock, this._viewerB.clock);
    }
  });

  this._viewerB.clock.onTick.addEventListener((clock) => {
    if (this._activeSource === 'B') {
      syncClock(clock, this._viewerA.clock);
    }
  });
}
```

### 4.5 图层控制

**两种模式：**

1. **独立模式**（默认）：每个 Viewer 独立切换影像图层和场景模式
2. **联动模式**：切换图层/场景模式时自动同步到对端

每个 Viewer 下方/面板内有独立的图层选择器和场景模式切换器。

### 4.6 实体联动 (EntitySync)

- 在两个 Viewer 中创建相同实体（不共享实例）
- 实体的增删改操作双向同步
- 选中状态联动：点击任一端实体，对端高亮同一实体
- 选中联动的方向也由 `activeSource` 决定

### 4.7 场景模式

- **两个 Viewer 均可独立切换** 3D / 2D / Columbus View
- 模式切换时自动调整相机同步策略（2D 相机参数与 3D 不同）
- 可选：联动切换模式

## 5. 交互功能

### 5.1 对等独立控制

- 每个 Viewer 可独立缩放、旋转
- 联动开关：可随时启用/禁用相机/时钟/图层联动
- 每个开关在工具栏中独立控制

### 5.2 状态栏（跟随活跃端）

- 坐标信息跟随当前活跃 Viewer 显示
- FPS 显示当前活跃 Viewer 的帧率
- 状态栏标注当前活跃 Viewer（"Viewer A" 或 "Viewer B"）

### 5.3 标注与测量

- 双屏同步绘制点/线/面标注
- 距离/面积测量工具（结果在双屏同步显示）

### 5.4 飞行定位

- 点击预设地点，双屏同步飞行（`camera.flyTo`）
- 搜索定位联动

## 6. 性能优化

| 策略 | 说明 |
|------|------|
| `requestRenderMode` | 仅在场景变化时渲染，降低 GPU 负载 |
| 节流同步 | 相机同步使用节流（16ms），避免每帧双重计算 |
| 按需加载 | 非活跃 Viewer 降低 `maximumScreenSpaceError` |
| 共享数据源 | 实体数据在内存层面共享，仅渲染对象独立 |

## 9. 旧版（主从模式）vs 新版（对等模式）对比

| 维度 | 旧版（主从） | 新版（对等） |
|------|-------------|-------------|
| Viewer 角色 | A=主控, B=从属 | A、B 完全对等 |
| 相机联动方向 | A→B 为主，B→A 需切换 | 谁操作谁是 Source，自动切换 |
| 时钟联动 | A→B 单向 | 双向，activeSource 决定方向 |
| 场景模式切换 | 仅 B 可切换 | A、B 均可独立切换 |
| 影像图层 | 统一切换 | 可独立切换，也可联动 |
| 状态栏追踪 | 仅追踪 A | 跟随当前活跃 Viewer |
| UI 标签 | "Viewer A · 3D" / "Viewer B · 3D / 2D" | "Viewer A" / "Viewer B"（对等） |
